ORIGINATION INTELLIGENCE PLATFORM
PATCH PLAN — CRM REAL / PIPELINE / ACTIVITIES / TASKS
VERSÃO TXT ÚNICA

OBJETIVO
Transformar a camada de CRM/Pipeline em um módulo real, persistido e operacional, conectado à inteligência da plataforma e ao frontend.

ESCOPO DO BLOCO
1. Persistência real de pipeline
2. Persistência real de activities
3. Persistência real de tasks
4. next_action persistida
5. Company Detail operacional
6. Pipeline Page operacional
7. Contratos backend/frontend alinhados

ALINHAMENTO COM O CÉREBRO MESTRE
Este patch plan existe para fechar a ligação entre descoberta, monitoramento, score, tese, ranking e CRM operacional, conforme a arquitetura institucional do projeto. O CRM/Pipeline é parte essencial do sistema porque a plataforma precisa conectar inteligência com execução comercial e transformar tese em pipeline real.

PARTE 1 — MODELAGEM DE DADOS

1. TABELA PIPELINE
Campos mínimos:
- id
- company_id
- stage
- owner
- next_action
- created_at
- updated_at

Observações:
- company_id precisa referenciar companies
- stage deve refletir o funil comercial institucional atual
- next_action deve ser um campo editável e também atualizável por inteligência

2. TABELA ACTIVITIES
Campos mínimos:
- id
- company_id
- type
- title
- description
- owner
- status
- due_date
- created_at
- updated_at

Observações:
- activity precisa poder representar follow-up, reunião, envio de material, validação interna etc.
- status pode começar simples: open / done / canceled

3. TABELA TASKS
Campos mínimos:
- id
- company_id
- title
- description
- owner
- status
- due_date
- created_at
- updated_at

Observações:
- task deve ser usada como item operacional de execução
- status pode começar simples: open / in_progress / done / canceled

4. ÍNDICES RECOMENDADOS
- índice por company_id em pipeline
- índice por company_id em activities
- índice por company_id em tasks
- opcional: índice por stage em pipeline
